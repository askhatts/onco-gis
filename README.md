# Онко-ГИС — Онкологический дашборд Абайской области

Интерактивная карта и аналитический дашборд: скрининг онкологических заболеваний по медицинским организациям, эпидемиологические показатели по районам, хороплет и динамика по периодам.

---

## Что умеет система

- Карта Абайской области с районами (хороплет по эпид. показателям) и МО (кружки по охвату скрининга)
- 7 эпидемиологических показателей: заболеваемость, смертность, ранняя диагностика, запущенность, выживаемость и др.
- Скрининг РМЖ / КРР / РШМ: рейтинг МО, динамика, сводка
- Фильтры: год, квартал, район, слои карты
- Обновление данных через веб-панель администратора (без программирования)

## Установка на новый компьютер (пошагово для не-программиста)

### Шаг 1 — Установить Python

1. Зайдите на сайт **python.org/downloads**
2. Нажмите большую жёлтую кнопку **«Download Python 3.x.x»**
3. Запустите скачанный файл
4. **ВАЖНО:** на первом экране установщика поставьте галочку **«Add Python to PATH»**
5. Нажмите «Install Now» и дождитесь окончания

> Проверка: откройте Командную строку (Win+R → `cmd` → Enter) и введите `python --version`. Должно написать что-то вроде `Python 3.12.0`

### Шаг 2 — Установить Git

1. Зайдите на сайт **git-scm.com/download/win**
2. Скачайте и запустите установщик
3. Везде нажимайте «Next» (настройки по умолчанию подойдут)

> Проверка: в Командной строке введите `git --version`

### Шаг 3 — Скачать проект

Откройте Командную строку и введите (замените `<адрес>` на ссылку из GitHub):

```
git clone https://github.com/Askhatts/onco-gis.git
cd onco-gis
```

Либо на GitHub нажмите **Code → Download ZIP**, распакуйте архив.

### Шаг 4 — Установить библиотеки Python

В Командной строке, находясь в папке проекта (`cd onco-gis`):

```
pip install fastapi "uvicorn[standard]" python-multipart openpyxl
pip install geopandas shapely pyogrio
```

> Установка займёт 1–3 минуты. Если `pip` не найден — перезапустите Командную строку после установки Python.

### Шаг 5 — Разместить файлы данных

Скопируйте следующие файлы в папку **`raws\`** внутри проекта:

| Файл | Что содержит |
|------|-------------|
| `РМЖ (1)бн.xlsx` | Скрининг рака молочной железы |
| `КРР (1)бн.xlsx` | Скрининг колоректального рака |
| `РШМ (1)бн.xlsx` | Скрининг рака шейки матки |
| `координаты МО бн.xlsx` | Список МО с координатами |
| `template_epidemiology-2.xlsx` | Эпидемиологические показатели |

> Папка `raws\` уже создана в проекте. Просто скопируйте файлы туда.

### Шаг 6 — Обработать данные

В Командной строке (в папке проекта):

```
python scripts/process_screening.py
python scripts/process_epidemiology.py
```

Каждая команда выполняется 10–30 секунд. В конце должно написать что-то вроде:
```
Wrote screening_rmzh.json (60 records...)
Wrote epidemiology.json (104 records)
```

### Шаг 7 — Запустить сервер

```
python -m uvicorn backend.main:app --host 127.0.0.1 --port 8080 --reload
```

Сервер запустился, если в Командной строке появилось:
```
INFO:     Application startup complete.
INFO:     Uvicorn running on http://127.0.0.1:8080
```

### Шаг 8 — Открыть встроенную версию в браузере

Откройте браузер (Chrome, Firefox, Edge) и введите адрес:

```
http://localhost:8080/app.html
```

### Шаг 9 — Современный интерфейс React/Vite (опционально)

Если хотите запускать новую версию интерфейса в режиме разработки:

```
cd frontend
npm install
npm run dev
```

После этого откройте:

```
http://localhost:5173
```

> Для Vite backend должен продолжать работать локально на `http://localhost:8080`.

### Шаг 10 — Запуск через Docker Compose (опционально)

Если на компьютере уже установлен Docker Desktop, можно поднять frontend и backend контейнерами:

```
docker compose up --build
```

После запуска:

- интерфейс: `http://localhost`
- API backend: `http://localhost:8000`

## Обновление данных без перезапуска

### Через панель администратора (рекомендуется)

1. Нажмите кнопку **⚙ Данные** в правом верхнем углу
2. Введите пароль: `admin2024`
3. Вкладка **Эпидемиология** — загрузить новый `template_epidemiology-2.xlsx`
4. Вкладка **Скрининги** — выбрать тип (РМЖ / КРР / РШМ) и загрузить файл
5. Вкладка **Пересчитать из raws/** — если вы вручную скопировали файлы в папку `raws\`, нажмите «Пересчитать все данные»

### Через командную строку

```
# Скопировать новый файл в raws\, затем:
python scripts/process_screening.py
python scripts/process_epidemiology.py
```

Страница обновится автоматически (нажмите Ctrl+F5 если не обновилась).

## Формат файлов данных

### Эпидемиология (`template_epidemiology-2.xlsx`)

Файл содержит один лист. Первая строка — заголовки. Начиная со второй строки — данные:

| Столбец | Заголовок | Пример |
|---------|-----------|--------|
| A | Район | Жарминский район |
| B | Год | 2025 |
| C | Заболеваемость | 245.3 |
| D | Смертность | 89.1 |
| E | Смертность_к_заболеваемости_% | 36.3 |
| F | Ранняя_диагностика_% | 61.5 |
| G | число впервые выявленных больных ЗН 0-I ст | 28 |
| H | Запущенность_% | 14.2 |
| I | Выживаемость_5лет_% | 57.8 |

Допустимые названия районов:
```
г. Семей, Абайский район, Аксуатский район, Аягозский район,
Бескарагайский район, Бородулихинский район, Жанасемейский район,
Жарминский район, Кокпектинский район, г. Курчатов,
Маканчинский район, Урджарский район
```

### Скрининг (`РМЖ (1)бн.xlsx` и аналогичные)

Файл в стандартном формате МЗ РК (исходный от МИАЦ). Скрипт ищет в нём строки с данными по пациентам, автоматически определяет МО, год и квартал по дате окончания скрининга.

### Координаты МО (`координаты МО бн.xlsx`)

| Столбец A | Столбец B |
|-----------|-----------|
| Полное название МО | Координаты (широта, долгота через запятую или пробел) |

Пример: `Учреждение "Поликлиника "Хаким"` → `50.366978, 80.259697`

## Смена пароля администратора

1. Откройте Командную строку и введите:
   ```
   python -c "import hashlib; print(hashlib.sha256(b'ВАШ_НОВЫЙ_ПАРОЛЬ').hexdigest())"
   ```
2. Скопируйте полученный хеш
3. Перед запуском сервера задайте переменную окружения:
   ```
   # Windows (Командная строка)
   set ADMIN_PASSWORD_HASH=вставить_хеш_сюда
   python -m uvicorn backend.main:app --host 127.0.0.1 --port 8080 --reload
   ```

## Частые проблемы

| Проблема | Причина | Решение |
|----------|---------|---------|
| Ошибка `WinError 10013` при запуске | Порт 8080 занят | Замените `--port 8080` на `--port 8090` и откройте `http://localhost:8090/app.html` |
| `pip` не найден | Python установлен без PATH | Переустановите Python, отметив «Add Python to PATH» |
| Карта не загружается | Нет интернета для фоновой карты | Нужен доступ к `basemaps.cartocdn.com` |
| Данные не обновились | Браузер закэшировал | Нажмите **Ctrl+F5** в браузере |
| `404 — run scripts first` | JSON-файлы не созданы | Выполните шаг 6 (запуск скриптов обработки) |
| Загрузка в AdminPanel даёт 401 | Неверный пароль | По умолчанию: `admin2024` |
| Районы не отображаются на карте | Нет файла `abay_districts.geojson` | Запустите `python scripts/filter_geodata.py` (нужен файл `geodata/KAZ_OSM_BORDER_LVL3.shp`) |

## Структура проекта

```
onco-gis/
├── backend/
│   ├── main.py              # Веб-сервер (FastAPI)
│   ├── Dockerfile           # Контейнер backend
│   └── requirements.txt     # Зависимости сервера
├── frontend/
│   ├── src/                 # Современный интерфейс React/Vite
│   ├── Dockerfile           # Контейнер frontend
│   ├── package.json         # Frontend-зависимости
│   └── public/
│       ├── app.html         # Главная страница
│       ├── app.jsx          # Встроенная legacy-версия интерфейса
│       ├── abay_districts.geojson   # Границы районов
│       └── *.json           # Данные (создаются скриптами)
├── geodata/
│   └── KAZ_OSM_BORDER_LVL3.shp    # Исходник границ (Казахстан)
├── raws/                    # Сырые Excel-файлы (НЕ публикуются в git)
├── scripts/
│   ├── process_screening.py      # Обработка скрининга
│   ├── process_epidemiology.py   # Обработка эпидемиологии
│   ├── filter_geodata.py         # Создание GeoJSON из shapefile
│   └── requirements.txt          # Доп. зависимости для geodata-скриптов
├── docker-compose.yml            # Совместный запуск контейнеров
├── .gitignore
└── README.md
```
